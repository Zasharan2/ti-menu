Overview

I have coded too many TI-84 applications to count, and often while coding I find myself repeating some of the same patterns over and over again. Namely, I face a lot of repetition when I am creating menu applications. Therefore, I created this interface to simplify some of these repeating patterns slightly.

TI-MENU is an abstraction of TI-BASIC, with simplified commands to make creating menu interfaces easier. It is a brace {} based, newline sensitive language. Here is how it works:


Instructions

TITLE: This instruction takes one argument: the title of the menu select. It has braces. Within the braces, only OPTION instructions may be placed.

OPTION: This instruction takes one argument: the name of the option. It has braces. All instructions placed within its braces represent the instructions to be executed upon selecting that option.

INPUT: This instruction takes at least one argument: the text to display when prompting input (optional), and the variable to store input into. It does not have braces.

DISPLAY: This instruction takes multiple arguments: The texts or variables to display. It does not have braces.

WAIT: This instruction takes no arguments. It does not have braces. It will pause program execution, then continue upon the user's pressing of the "Enter" key.

STOP: This instruction takes no arguments. It does not have braces. It will stop program execution.

WAITSTOP: This instruction takes no arguments. It does not have braces. It will pause program execution, then stop upon the user's pressing of the "Enter" key.

NOTE: This instruction takes one argument: the text to be written as a note. It does not have braces. It will display the note text (with multiple pages if necessary), and wait for the user to press enter before stopping.

IF: This instruction takes one argument: the condition. It has braces. All instructions placed within its braces are executed only if the condition is met.

ELSE: This instruction takes no arguments. It has braces. All instructions placed within its braces are executed only if the prior IF condition was not met.

FOR: This instruction takes three arguments: the variable to iterate, the lower bound, and the upper bound. It has braces. All instructions placed within its braces are executed as the variable iterates from the lower bound to the upper bound.

WHILE: This instruction takes one argument: the iteration condition. It has braces. All instructions placed within its braces are executed while the condition is met.

MATH: This instruction takes one argument: the mathematical expressions and assignment to be carried. It does not have braces.

CLEAR: This instruction takes no arguments. It does not have braces. It will clear the screen.

INJECT: This instruction takes no arguments. It has braces. All TI-BASIC code placed within its braces will be directly inserted to the program as is.
